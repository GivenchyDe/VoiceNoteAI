# 基于Web的语音识别笔记系统的设计与实现

## 一、软件工程

- 立项（确定目标）：开发一款基于 Web 的语音识别笔记系统，核心目标是通过语音识别技术实现语音到文字的快速转换，结合 Web 端便捷的交互体验，为用户提供高效的笔记创建、管理、检索能力，解决传统文字笔记录入效率低的问题。
- 可行性研究（技术、经济 - ROI、社会：可行性研究报告）
  - 技术可行性：当前 Web 前端技术（Vue3、Web Audio API）可实现语音录制 / 上传，百度智能云、讯飞开放平台等提供成熟的语音识别 API，后端 SpringBoot3.0 框架能稳定支撑业务逻辑，MySQL、Redis 等数据库技术可满足数据存储与缓存需求，技术栈均为成熟主流方案，具备落地条件。
  - 经济可行性：开发阶段可复用开源框架降低成本，语音识别 API 具备免费试用额度，中小企业级部署服务器成本低；投入使用后，可通过提升用户笔记记录效率降低时间成本，ROI 预计 6-12 个月实现正向收益。
  - 社会可行性：符合用户便捷化、高效化的笔记记录需求，遵循数据隐私保护法规，保障用户音频和笔记数据安全，无社会伦理与合规风险。
- 需求分析（功能需求、非功能需求分析）
- 系统设计（技术架构，数据库设计）
- 系统实现（环境搭建，服务端开发，前端开发）
- 系统测试（测试方法，测试用例，测试结论）
- 系统部署
- 运维

## 二、需求分析

### 2.功能需求分析:

主要根据系统的使用群体，提取出不同的角色（用户，管理员），根据角色界定角色的能力范围（这个角色可以做什么）

#### 2.1 用户角色 - Web 端

认证模块：包括用户注册、登录、密码找回等基本功能，支持手机号 / 邮箱 + 验证码登录。

语音录制 / 上传模块：支持实时语音录制（基于 Web Audio API），或上传本地音频文件（支持 mp3、wav、m4a 等主流格式）。

语音识别模块：调用第三方语音识别 API（讯飞 / 百度）将语音转换为文字，支持普通话、英语、主流方言，可实时显示识别进度，支持识别结果一键修正。

笔记管理模块：对识别后的文字内容进行笔记创建、编辑、删除、归档，支持笔记分类（工作、生活、学习等）、标签管理，支持笔记内容的富文本编辑。

语音检索模块：支持通过语音输入或文字关键词检索笔记内容，可按创建时间、分类、标签筛选检索结果。

笔记导出模块：支持将笔记导出为 txt、markdown、PDF 等格式，支持单条 / 批量导出。

个人中心：个人信息管理（头像、昵称、联系方式）、密码修改、识别记录查看、API 使用额度查询、笔记备份 / 恢复。

反馈建议：对系统识别准确率、功能体验、音频兼容性等问题进行反馈，支持上传截图 / 音频附件。

#### 2.2 管理员角色 - Web 后台管理端

系统登录：管理员提供账号密码 + 验证码登录系统，支持登录日志查看。

首页仪表盘：展示系统的核心统计数据卡片（用户数、累计识别次数、音频文件总数、笔记总数、平均识别准确率等）；通过 Apache-Echarts 实现可视化图表展示（新增用户趋势、日 / 周识别量统计、音频文件格式分布、笔记分类占比）。

用户管理：对系统用户进行 CRUD 操作，支持按手机号 / 昵称检索用户，可禁用 / 启用用户账号，查看用户的识别记录、笔记列表。

识别日志管理：查看所有用户的语音识别记录（用户 ID、音频文件、识别耗时、识别结果、创建时间），支持按时间、用户、识别状态筛选，支持日志导出。

反馈管理：对用户提交的反馈建议进行管理（查看详情、标记处理状态、回复用户、删除无效反馈）。

系统管理：

- API 配置：管理第三方语音识别 API 的密钥、调用地址，支持多引擎切换；
- 系统参数：设置音频文件大小限制、识别超时时间、支持的音频格式；
- 管理员账号管理：新增、编辑、删除管理员，分配不同操作权限；
- 公告管理：发布系统维护、功能升级、API 额度调整等公告，支持编辑、删除、置顶。

### 3.2 数据库设计

#### 3.2.1 概念模型

核心实体及关系：

- 用户（User）：包含用户基本信息，一对多关联笔记、音频文件、识别日志、反馈；
- 笔记（Note）：关联所属用户，一对多关联标签；
- 音频文件（AudioFile）：关联所属用户、对应的识别日志；
- 识别日志（RecognitionLog）：关联用户、音频文件，记录识别结果、耗时等；
- 反馈（Feedback）：关联所属用户；
- 管理员（Admin）：管理系统配置、用户、公告；
- 公告（Notice）：由管理员发布，面向所有用户。

## 四、系统实现

### 4.1 开发环境 & 开发工具

#### 4.1.1 开发环境

（1）服务端

核心开发语言：Java17

服务端框架：SpringBoot3.0

持久层框架：Mybatis-plus

文档生成工具：Knife4J（接口文档）

缓存数据库：Redis（用户 Token、识别结果缓存）

接口网关：Spring Cloud Gateway（可选）

（2）前端（用户端 + 管理端）

JS 框架：Vue3

UI 框架：ElementPlus

网络请求：Axios

可视化技术：Apache-Echarts

语音处理：Web Audio API（录制）、FileReader（文件解析）

（3）第三方服务

语音识别 API：讯飞开放平台语音识别 API / 百度智能云语音识别 API

文件存储：本地文件系统（测试阶段）/ 阿里云 OSS（生产阶段）
